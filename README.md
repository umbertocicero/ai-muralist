# 🎨 AI Muralist

> An autonomous AI agent wanders a tiny 3D city and spontaneously paints murals on the walls.
> Every mural is a unique SVG artwork generated in real time by **Claude**. It runs forever, in the browser, with no human input.

![status](https://img.shields.io/badge/status-experiment%20%E2%84%96001-ff6b35) ![cost](https://img.shields.io/badge/hosting-%240%2Fmo-2d6a4f)

A little orange figure named **MURO** walks the streets, finds a blank wall, imagines something, and paints it. It's hypnotic, shareable, and a tiny demonstration of autonomous creative AI.

---

## ✨ What it does

- 🏙️ A 3D miniature city (16 buildings, roads, sidewalks, lampposts) rendered with **Three.js**
- 🚶 An autonomous agent with AABB collision, wall-sliding and steering — it never walks through buildings
- 🧠 A **5-state agent brain**: `WANDERING → MOVING_TO_WALL → THINKING → PAINTING → CONTEMPLATING`
- 🖌️ Each mural is a **fresh SVG** generated by Claude across 8 rotating art styles (Bauhaus, Op-Art, Cosmos, Graffiti…)
- 💬 A poetic "inner monologue" for every mural
- 🎥 Slow cinematic orbit camera — a passive, ambient experience

## ⚡ Quick start (local, offline demo)

You don't need any API key to *see it run*. Out of the box `WORKER_URL` is `null`, which puts the app in **offline demo mode** — murals are generated by a local procedural fallback.

```bash
# from the project folder
python3 -m http.server 8000
# open http://localhost:8000
```

Or just open `index.html` directly in a browser. Watch MURO walk, think, and paint.

To use **real Claude-generated murals**, deploy the Worker (below) and set `WORKER_URL` in `index.html`.

## 🚀 Full deploy (free, ~10 minutes)

The architecture:

```
Browser ──▶ Cloudflare Pages (index.html, static, free)
        └─▶ Cloudflare Worker (/api proxy, free) ──▶ Anthropic API
                     └─▶ Cloudflare KV (rate limiting, free)
```

### Prerequisites

- A free [Cloudflare account](https://dash.cloudflare.com/sign-up)
- An [Anthropic API key](https://console.anthropic.com/) with billing enabled
- Node.js installed

### 1 · Install & log in to Wrangler

```bash
npm install -g wrangler
wrangler login
```

### 2 · Create the rate-limit KV namespace

```bash
wrangler kv:namespace create RATE_LIMIT_KV
```

Copy the returned `id` into **`wrangler.toml`**, replacing `REPLACE_WITH_YOUR_KV_NAMESPACE_ID`:

```toml
[[kv_namespaces]]
binding = "RATE_LIMIT_KV"
id = "abc123...your-real-id"
```

> Don't want rate limiting? Delete the `[[kv_namespaces]]` block — the Worker still works.

### 3 · Deploy the Worker

```bash
wrangler deploy
```

Wrangler prints your Worker URL, e.g.:

```
https://ai-muralist-proxy.<your-account>.workers.dev
```

### 4 · Set the API key (secret, never in code)

```bash
wrangler secret put ANTHROPIC_API_KEY
# paste your sk-ant-... key when prompted
```

### 5 · Point the app at your Worker

Open **`index.html`** and set the config constant near the top of the `<script>`:

```js
const WORKER_URL = 'https://ai-muralist-proxy.<your-account>.workers.dev';
```

### 6 · Deploy the front-end to Cloudflare Pages

**Option A — CLI (fastest):**

```bash
wrangler pages deploy . --project-name ai-muralist
```

**Option B — GitHub integration (auto-deploy on push):**

1. Push this repo to GitHub
2. Cloudflare Dashboard → **Workers & Pages → Create → Pages → Connect to Git**
3. Pick the repo. **Build command:** *(leave empty)*. **Output directory:** `/`
4. Deploy

Your site is live at `https://ai-muralist.pages.dev` 🎉

## 🔌 Alternative: same-origin `/api/claude` (no CORS, no separate URL)

Instead of a standalone Worker, you can use **Pages Functions** so the API lives at the same origin as the site. Create `functions/api/claude.js`:

```js
// functions/api/claude.js
import worker from '../../worker.js';
export const onRequest = (context) => worker.fetch(context.request, context.env);
```

Add your secret in **Pages → Settings → Variables → Secrets** as `ANTHROPIC_API_KEY`, then set in `index.html`:

```js
const WORKER_URL = '/api/claude';
```

This avoids cross-origin requests entirely.

## 🛠️ Configuration cheatsheet

All knobs live at the top of the `<script>` in `index.html`:

| Constant | Default | Meaning |
|---|---|---|
| `WORKER_URL` | `null` | Worker endpoint. `null` = offline demo mode |
| `MODEL` | `claude-sonnet-4-6` | Claude model used for generation |

And in `index.html` further down you can tune `MOVE_SPEED`, building layout (`positions`), camera distance/height, fog, and the mural prompt itself (`buildPrompt`).

## 🧩 How it works

1. The agent **wanders** to random reachable points (collision-checked).
2. After a while it **picks a free wall slot** whose approach point is unobstructed.
3. It walks to the wall and enters **THINKING** — firing one API call that returns both a poetic thought *and* the mural SVG.
4. The thought appears; the agent **paints** (animated arm), then the SVG is rasterized to a `CanvasTexture` and applied to a plane on the wall.
5. When **every** wall is painted, it enters **CONTEMPLATING** and just roams.

Each building exposes 4 wall slots (N/S/E/W) → up to **64 murals** per session before contemplation.

See [`PROMPT.md`](./PROMPT.md) for the complete spec.

## 🐛 Troubleshooting

| Symptom | Fix |
|---|---|
| Murals never appear, console shows `rate_limited` | Normal if reloading fast — the Worker limits 1 req/8s per IP |
| `Server missing ANTHROPIC_API_KEY` | Run `wrangler secret put ANTHROPIC_API_KEY` |
| CORS errors in console | Use the same-origin `/api/claude` option, or confirm the Worker CORS headers |
| Blank/garbled mural | The model occasionally returns invalid SVG; the slot is released and retried later |
| Nothing renders at all | Check the Three.js CDN loaded (network tab) and WebGL is enabled |
